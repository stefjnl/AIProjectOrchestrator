Frequently Asked Questions

Find answers to common questions about the AI Project Orchestrator

Getting Started

What is AI Project Orchestrator?

AI Project Orchestrator is an enterprise-grade platform that automates software development workflows using multiple AI models. It guides your project through a 5-stage pipeline: Requirements Analysis, Project Planning, User Stories, Prompt Generation, and Code Generation.

How do I create my first project?

Click "Create Project" in the navigation, fill out the project form with your requirements, and submit. The system will automatically start the workflow process.

What AI models are supported?

We support Claude, OpenRouter, and LM Studio (local). You can choose your preferred model or let the system auto-select the best available option.

Is there a cost to use this platform?

The platform itself is free, but AI model usage may incur costs depending on your chosen provider and usage volume.

Workflow Process

How does the 5-stage workflow work?

Each stage builds upon the previous one: 1) Requirements Analysis understands your needs, 2) Project Planning creates a development roadmap, 3) User Stories breaks down features, 4) Prompt Generation creates implementation prompts, and 5) Code Generation produces the final code.

Do I need to approve each stage?

Yes, human approval is required at each stage to ensure quality and alignment with your vision. You can approve, reject with feedback, or request modifications.

How long does each stage take?

Timing varies by project complexity and model availability. Simple projects might complete in minutes, while complex enterprise applications could take several hours across all stages.

Can I modify the generated content?

Absolutely! You can edit any generated content before approval, provide feedback for regeneration, or manually adjust the final outputs.

Technical Questions

What technologies are used?

The platform is built with .NET 9, Entity Framework Core, PostgreSQL, and modern web technologies (HTML5, CSS3, vanilla JavaScript). It uses Clean Architecture principles for maintainability.

How is my data secured?

All data is encrypted in transit and at rest. We follow enterprise security standards including secure API endpoints, input validation, and regular security audits.

Can I run this locally?

Yes! The entire system can be deployed locally using Docker. Check our GitHub repository for deployment instructions and configuration options.

What file formats are supported?

The system works with various formats including Markdown for documentation, JSON for data exchange, and standard code files for implementation.

Support & Troubleshooting

What if the AI generates incorrect content?

You can reject the content with specific feedback, and the AI will regenerate based on your input. You can also manually edit the content before approval.

How do I report bugs or issues?

Use the system analysis page to check system health, or create an issue in our GitHub repository with detailed information about the problem.

Can I suggest new features?

Absolutely! We welcome feature requests through our GitHub issues. Please describe your use case and how the feature would improve your workflow.

What browsers are supported?

We support all modern browsers including Chrome, Firefox, Safari, and Edge. For the best experience, use the latest version of your preferred browser.
